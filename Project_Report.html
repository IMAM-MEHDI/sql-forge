1. ABSTRACT

SQLForge is a modern, full-stack web application designed to revolutionize how students learn SQL. Unlike traditional multiple-choice platforms, SQLForge provides a Live SQL Sandbox where users write and execute real PostgreSQL queries against isolated schemas. The platform integrates gamification through a global leaderboard, personalized progress tracking, and 5 progressive levels of difficulty. Built with a FastAPI backend and a React frontend, it features a highly secure execution engine that prevents destructive operations through restricted database users and transaction rollback patterns. The system includes an administrative suite for designing custom levels, including schema definitions and automated win-condition validation using JSON result sets.

2. INTRODUCTION

Learning SQL effectively requires hands-on practice, but setting up local databases can be a barrier for beginners. SQLForge removes this barrier by providing an instantly accessible, browser-based environment for database experimentation.

Objectives:

  • Provide a secure, isolated PostgreSQL sandbox for query execution.
  • Automate the validation of SQL results against "ground truth" data.
  • Enhance student engagement through a competitive global leaderboard.
  • Enable instructors to design complex challenges with custom schemas and seed data.
  • Track individual learning progress across different SQL topics (SELECT, WHERE, JOINs, Aggregations).

3. SYSTEM ARCHITECTURE

The system follows a modern decoupled architecture:

  • Frontend: React 18 with Vite, featuring Monaco Editor for a professional IDE experience and Lucide React for iconography.
  • Backend: FastAPI (Python 3.14), utilizing SQLAlchemy 2.0 for ORM and Psycopg3 for high-performance database communication.
  • Database: PostgreSQL 18, utilizing a multi-role security model to isolate user execution from the main application state.
  • Security: JWT (JSON Web Tokens) for stateless authentication and Role-Based Access Control (RBAC).

6. SECURITY IMPLEMENTATION

To prevent malicious SQL injection or destructive commands (like DROP TABLE), SQLForge implements three layers of security:

  1. Restricted User: Queries are executed by a sandbox_user who only has SELECT and INSERT permissions on a specific schema.
  2. Transaction Isolation: Every user query is wrapped in a BEGIN...ROLLBACK block. This allows the system to check the result of an operation without permanently changing the database state.
  3. Schema Sandboxing: User queries are strictly confined to a temporary sandbox schema that is initialized and destroyed per-session.
