SQLForge: Project Setup & Migration

Tip: Both services are ready to be run locally for testing. Ensure you have Node.js and PostgreSQL installed.

1. Running the Application

Start the Backend (Python FastAPI)

cd backend
pip install -r requirements.txt
python -m uvicorn main:app --reload

Start the Frontend (React + Vite)

cd frontend
npm install
npm run dev

2. Migration to Another Computer

Follow these steps to move this project to a new machine:

Important: Do not copy node_modules or venv folders. These should be recreated on the new machine to ensure compatibility.

Step A: Transfer Files

Copy the project folder (excluding the folders mentioned above) to your new computer.

Step B: Database Re-Setup

Create the database and users in your new PostgreSQL instance:

CREATE DATABASE sql_game;
CREATE USER sql_user WITH PASSWORD 'your_secure_password';
GRANT ALL PRIVILEGES ON DATABASE sql_game TO sql_user;

Step C: Initialize Environment

In the backend/ folder of the new computer, create a .env file:

DATABASE_URL=postgresql://sql_user:your_secure_password@localhost:5432/sql_game
SECRET_KEY=generate_a_random_string

3. Database Initialization (Scripts)

Run these scripts in order to set up the core tables and the isolated sandbox environment:

I. Core Schema

Execute this as a superuser to create tables for users, levels, and submissions:

psql -U postgres -d postgres -f database\schema.sql

II. Sandbox Setup

This creates the restricted sandbox_user and the execution schema:

psql -U postgres -d postgres -f database\sandbox_setup.sql

Note: If you use pgAdmin or DBeaver, you can simply open these files and execute them in the SQL editor.

4. Seeding Data

To populate your new database with the SQL topics and level definitions:

cd backend
python seed.py
